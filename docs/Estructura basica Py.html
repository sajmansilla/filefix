Linea de Inicialización

Se utiliza generalmente solo en entornos Unix, esta línea permite la ejecución del script solo por el nombre.

Documentación del script

Aquí explicamos la utilidad del script o módulo que desarrollamos, realizaremos una especie de sumario sobre las variables globales y cualquier otra anotación que consideres relevante. Se puede acceder externamente a esta documentación simplemente con  nombreModulo.__doc__.

Módulos a Importar

Aquí importaremos todos los módulos necesarios para el código de nuestro script. Los módulos que aquí se definan se importan una vez (al cargar el módulo). Existen otros imports, los que se realizan dentro de funciones, estos solo se cargarán cuando se llame a dicha función.

Declaración de variables

Aquí declararemos las variables globales que utilizaremos en las múltiples funciones en este módulo.

Te recomendamos como buena práctica, evitar en lo posible utilizar variables globales en detrimento de variables locales, ya que mejora el rendimiento y utilización de memoria entre otras razones.

Declaración de Clases

Todas las clases que crearemos se deben declarar a continuación. Una clase está definida cuando el módulo que la contiene se importa y la sentencia class se ejecuta.  De igual forma cualquier documentación realizada aquí para resumir la funcionalidad de la clase se obtiene con nombreClase.__doc__.

Declaración de Funciones

Funciones que se declaren aquí serán accesibles externamente cuando se llaman de la forma nombreModulo.nombreFuncion(). Las función está definida cuando el módulo se importa y la sentencia def ejecutada. Al igual que la documentación de la clase, la documentación de la función se accede a través de la sentencia nombreFuncion.__doc__.

Cuerpo “main”

Todo el código en este nivel se ejecutará donde este módulo se importe o arranque como script. La gran mayoría de los scripts solo contendran una validación contra la variable __name__, que de validarse, se redirigirá al método o función determinado (Este método o función por lo general suele llamarse main(), para hacer constar que es la función principal, similar al main de java).
Como podemos ejecutar los códigos en Python de ambas formas (Cargarlo como módulo o ejecutarlo como script ), necesitamos conocer de que forma o a través de quien es ejecutado el módulo, por lo que la variable __name__ nos dará la solución.

  • __name__  Contendrá el nombre del módulo si este es importado por otro script.
  • __name__ Contendrá el valor literal ‘__main__’ cuando el módulo se ejecuta como un script.
